DTPR transparency website & signage — user workflow
Static template: edit configuration, preview locally, publish online, then produce and place physical signage linked to the live site.
Static site files → Public HTTPS URL → On-site signage & QR
Steps from setup to deployment
1
Copy the template
  • Keep full folder: HTML pages, css/, js/, assets/
  • No build tools required
›
2
Customize content
  • Edit js/config.js (project text, DTPR cards, survey, links, visuals)
  • Or use settings.html in the browser for the same fields
›
3
Preview locally
  • Run a static server (e.g. py -m http.server 8000)
  • Check home.html, index.html, signage.html
›
4
Deploy the website
  • Upload the whole site to a static host (GitHub Pages, Netlify, or web root)
  • Confirm HTTPS and bookmark the public URL (often …/home.html)
›
5
Prepare signage
  • Use signage.html for printable / sticker assets
  • Point QR codes to your live transparency URL
›
6
Install at the place
  • Mount signage where people naturally see it
  • Test QR and links on a phone

This workflow matches the bundled template: static disclosure pages, one configuration file for copy and DTPR-style cards, then hosting plus field signage so visitors can open the same information online. See README.md and deploy.html in this folder for detailed hosting options.
