Deployment Guide

Step-by-step instructions to host this DTPR platform online — for free, in under 10 minutes.

This is a static website — it runs entirely in the browser with no server or database required. You only need to upload the files to a web host. Below are three recommended options, ordered by simplicity.

1

Create a free GitHub account

Go to github.com and sign up if you don't already have an account.

2

Create a new repository

Click New → name it anything (e.g. my-dtpr-site) → set it to Public → click Create repository.

3

Upload your files

Inside the repository, click Add file → Upload files. Drag and drop all files and folders from your project directory (the folder containing home.html, index.html, css/, js/, assets/). Click Commit changes.

# Folder structure to upload: ├── home.html ├── index.html ├── settings.html ├── signage.html ├── deploy.html ├── css/ ├── js/ │ ├── config.js ← edit this to customise │ ├── app.js │ └── scripts.js └── assets/
4

Enable GitHub Pages

In your repository go to Settings → Pages. Under Source, select Deploy from a branch, choose main branch and / (root) folder. Click Save.

5

Access your live site

After ~60 seconds your site will be live at:
https://<your-username>.github.io/<repository-name>/home.html

Bookmark this URL and share it with the public.
6

Customise your content

Open js/config.js in any text editor. Change the project name, description, survey URL, etc. Upload the modified file back to GitHub (same upload flow). Changes go live in ~30 seconds.

Alternatively, use the Settings panel on the live site — changes persist in that browser via localStorage.
1

Create a Netlify account

Go to netlify.com → Sign up for free (you can use your GitHub account).

2

Drag-and-drop deploy

On your Netlify dashboard, scroll to the bottom — you'll see a deploy drop zone. Open your project folder in your file browser, select all files and folders, and drag them into the drop zone.

Netlify deploys instantly — your site is live in seconds with a URL like https://amazing-name-123.netlify.app.
3

Set the publish directory

Netlify may ask for a publish directory. Leave it as / (the root).

4

Custom domain (optional)

In Netlify → Domain settings, you can add a custom domain (e.g. dtpr.yourproject.org). Netlify also provides free HTTPS automatically.

5

Update content

To update, edit js/config.js then drag-and-drop the updated folder again, or connect a GitHub repo for automatic deploys on every commit.

1

Obtain web hosting

You need a host that serves static files over HTTP/HTTPS. Options include: Amazon S3 + CloudFront, Google Cloud Storage, Azure Static Web Apps, university IT hosting, or any shared hosting (cPanel). No PHP or Node.js required.

2

Upload files via FTP / SFTP

Use a client like FileZilla (free) to upload all project files to your public_html or equivalent web root directory.

Host: your-university.edu or hosting provider Username: your FTP username Password: your FTP password Port: 21 (FTP) or 22 (SFTP)
3

Set home.html as the index

If your host doesn't automatically serve home.html, create an index.html redirect file in the root:

<meta http-equiv="refresh" content="0; url=home.html">
4

Test your deployment

Visit your URL and check all pages. Open the browser console (F12) and look for any 404 errors (missing files). The most common issue is incorrect asset paths — make sure the assets/ folder was uploaded correctly.
